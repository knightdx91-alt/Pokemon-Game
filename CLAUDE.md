## What this game is

A FireRed-style overworld walker (pre-existing) that we turned into a playable
"Pokémon Crater"-style RPG: walk Kanto, wild encounters in grass, turn-based
battles, catching, party/leveling, NPCs, trainers — with a pixel-faithful
FireRed UI.

## Architecture (load order in `index.html`)

Engine: `src/engine/{input,camera,map,renderer}.js`
UI: `src/ui/{layout,controls,hud,startmenu}.js`
Legacy save/systems: `src/data/{save,achievements,factions}.js` (save.js slot

**RPG layer we added:**

## Data assets

- `data/pokedex.json` — 151 Kanto species (real Gen-1 typing; stats from
  `data/pokemon/base_stats.json`; derived catch rate + baseExp). Generated by a
  one-off node script (types table hand-authored).
- `data/sprites/pokemon/{N}.png` and `back/{N}.png` — Gen-1 front/back battle
  sprites (dex 1–151), fetched from PokeAPI/sprites. Bundled = self-contained.
- `data/ui/font_normal.png` + `data/ui/font.json` — FireRed Latin font atlas
  (index1=fg colored `#484850`, index2=shadow `#d0d0c8`, transparent bg) + charmap
  with measured proportional widths. Extracted from pret `graphics/fonts/latin_normal.png`
  + `charmap.txt` (each glyph = 16×16 cell, byte-indexed: cell = byte%16, byte//16).
- `data/encounters/kanto.json` — pret wild encounter format
  (`wild_encounter_groups[0].encounters[]`, keyed by `MAP_*`, `land_mons.mons[]`).
- Maps/layouts/tilesets under `data/{maps,layouts,tilesets}` (extracted).

## Key engine facts / gotchas

- **Grass detection:** `GameMap.isGrass(x,y)` → tileset `behaviors[metatileId] === 2`
  (MB_TALL_GRASS). Tileset JSONs (`data/tilesets/*.json`) have a `behaviors` array
  indexed by metatile id; loaded/cached in `map.js` on layout load.
- **Grass walkability fix:** extracted collision marks grass tiles as blocked, so
  `isWalkable` was patched to return true for grass — otherwise encounters can
  never fire. (map.js)
- **Encounters** only wired for Kanto maps (encounter table keyed by `MAP_*` id).
- **Battle input:** rising-edge on `GameInput.state` (a/b/dpad). A `busy` flag was
  removed earlier because it deadlocked message advancement — do NOT reintroduce
  input-gating that blocks message advance.
- **Battle canvas:** 240×160, CSS-scaled with `image-rendering: pixelated`. All text
  via `GameFont.draw(ctx, text, x, y, scale)`. `#battle-overlay` centers `#battle-canvas`.
- **Interactions:** press A facing a tile → NPC dialogue / sign / item-ball pickup
  (persisted in `GameParty.state.collected`) / trainer battle (`defeatedTrainers`
  remembered). Trainer party generated from map's encounter species scaled to
  player level.

## Dev / test workflow

- Serve: `python3 -m http.server 8099` from repo root.
- Headless test: node CommonJS with
  `require('/opt/node22/lib/node_modules/playwright/index.js')`, launch chromium
  with `executablePath:'/opt/pw-browsers/chromium'`.
- Fresh game in tests: `localStorage.clear()` then reload (note: an `addInitScript`
  that clears storage will also wipe on every reload — clear once via `evaluate`).
- Simulate key: `keyboard.down(k)`, wait ~60ms, `keyboard.up(k)` (a quick
  `press()` can be missed by the rAF input sampler). Keys: z=A, x=B, Enter=A,
  arrows=dpad. Switch-menu after a faint needs arrow nav, not just mashing A.
- Player position exposed as `window.GamePlayer` for test setup.
